# 二次核验

## 用途

二次核验，是为了防止一篇顺滑的小作文变成错误的投资结论。

模糊的正确很重要，但事实错误和逻辑断裂必须先干掉。数据不核验，后面说得再漂亮，都是空中楼阁。

## 三层核验

### 第一层：事实核验

事实核验是整个投资流程里最重要的规则。它必须发生在结论、估值区间、仓位计划和风险标签之前。

所有数字在用于投资结论前都必须核验。分析上市证券时，先用程序化数据源或可执行脚本拉取最新数据，再用网络搜索、交易所公告、公司公告、财报或权威公开来源交叉验证。

未核验数据不能当证据。关键数字无法核验时，要明确标记为未核验，并从估值和仓位测算中剔除；必要时只能给框架判断。

必须核验：

1. 最新股价、市值、市盈率、市净率、股息率。
2. 最新年报、半年报、季报。
3. 关键公告：分红、回购、定增、减持、股权激励、重大投资、诉讼、监管处罚。
4. 原始政策文件和发布机关。
5. 商品价格、产量、库存、开工率。
6. 可比公司定义和会计口径是否一致。
7. 估值使用的前瞻盈利预测，包括研报来源、机构、发布日期、预测期间、假设和置信度。

优先使用一手来源：交易所公告、公司公告、政府网站、统计局、行业协会、官方数据库。

### 数据冲突规则

1. 程序拉取数据和网页搜索数据不一致时，优先使用最新的程序拉取数据。
2. 如果程序源明显过期、失败、不完整或内部矛盾，则降级使用最新权威公告、交易所文件或公司公告。
3. 影响估值和仓位的数字，必须写明日期、报告期或数据时间戳。
4. 重大数据冲突无法解决时，不强行给估值结论，只标记待核验，并给框架判断。
5. 预测数据冲突时，按发布时间、机构可信度、与已核验财务数据的一致性、假设透明度选择高置信度数据；没有可信研报时，用常规算法估算，并标明“内部估算”。

### 第二层：逻辑核验

检查逻辑链是否成立：

```text
结构性问题是否存在？
→ 行业约束是否真实？
→ 公司是不是解决问题的工具？
→ 关键变量是否真的影响利润？
→ 财报是否印证利润路径？
→ 估值是否提供安全边际？
→ 仓位是否匹配不确定性？
```

只要有一环断了，结论就要降级。

### 第三层：证伪核验

必须问：

1. 什么事实出现，说明这个逻辑错了？
2. 关键变量反向变化，利润会掉多少？
3. 如果分红下降，安全绳还在不在？
4. 如果政策不执行，股票还便宜吗？
5. 如果产能延期，当前市值还能不能支撑？
6. 如果商品价格下跌，是不是周期顶部陷阱？
7. 市场低估它，是不是因为它真的弱？

## 估值核验

输出估值区间前，必须检查：

- 5–10 年估值分位。
- 商业模式相近的可比公司。
- 本年度悲观、中性、乐观三档前瞻市盈率。
- 研报预测来源和置信度；没有研报时写明内部估算假设。
- 派息率和现金流覆盖。
- 利润对关键变量的敏感性。
- 悲观情景下的下行保护。

## 仓位核验

给仓位建议前，必须检查：

1. 单一标的上限。
2. 和已有持仓的相关性。
3. 是否保留现金等机会。
4. 这笔钱是不是亏得起。
5. 判断错了会不会造成多年伤害。
